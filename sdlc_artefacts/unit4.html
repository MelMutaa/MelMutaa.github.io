What is ReDOS and what part do ‘Evil Regex’ play?

ReDOS (Regular Expression Denial of Service) is a type of Denial of Service (DoS) attack where an attacker exploits the inefficiencies of regular expressions (regex) to overwhelm the target system. The core issue in ReDOS attacks is that poorly designed regular expressions—often referred to as "Evil Regex"—can lead to excessive backtracking. This occurs when a regular expression engine tries multiple permutations to match a pattern, consuming significant CPU time and resources.

According to Larson (2018), an "Evil Regex" is a regular expression with certain characteristics that make it vulnerable to excessive backtracking. These characteristics include:

  • Nested quantifiers (e.g., a+ inside another quantifier, such as (a+)+)
  • Alternation with overlapping patterns (e.g., (a|aa) where both alternatives start with the same characters)
  • Greedy quantifiers that consume as much input as possible before backtracking to find matches

In a ReDOS attack, attackers send specially crafted input that triggers this excessive backtracking, potentially crashing the system or significantly slowing down performance.

What are the common problems associated with the use of regex? How can these be mitigated?

While regex is a powerful tool for pattern matching, it presents several common problems:

  • Complexity and Readability: Regex can become complex and difficult to understand, especially for more intricate patterns. This complexity increases the likelihood of bugs or inefficiencies in the expression.
  • Performance: Poorly written regex can cause performance bottlenecks. Certain regex patterns, particularly those with overlapping or ambiguous rules, can lead to excessive backtracking, consuming more CPU resources than expected (Larson, 2018).
  • Security Vulnerabilities: Improper use of regex may open up vulnerabilities like ReDOS, where attackers can intentionally exploit inefficiencies in regex evaluation.

Mitigation strategies include:

  • Limit the Input Size: Avoid applying regex to large, untrusted input unless strictly necessary. Input validation and length limits can help prevent ReDOS attacks.
  • Precompile Regular Expressions: Precompiling regex patterns allows them to be optimized by the engine and avoids recomputing the same pattern repeatedly, which can improve performance.
  • Use Safe Libraries: Some regex libraries are designed to be more resistant to ReDOS vulnerabilities by imposing limits on backtracking or using more efficient algorithms for pattern matching (Weidman, n.d.).
  • Testing and Profiling: Thoroughly test regular expressions, especially those used in production systems, to identify potential performance issues. Profiling tools can help detect problematic expressions.

How and why could regex be used as part of a security solution?

Regular expressions can play a vital role in security by enabling pattern-based input validation, which helps prevent common web vulnerabilities such as SQL injection, cross-site scripting (XSS), and command injection. When used effectively, regex can:

  • Sanitize Input: Regex is commonly used to validate and sanitize user input by ensuring that the input conforms to expected patterns (e.g., validating email addresses, URLs, or phone numbers). This can help prevent malicious data from being processed by the application (Weidman, n.d.).
  • Detect Malicious Patterns: Regex can be employed in intrusion detection systems (IDS) or security filters to detect patterns of malicious behaviour, such as signature-based detection of malware or filtering of known attack vectors.

However, it’s crucial to ensure that the regex patterns used in security solutions are efficient and resistant to exploitation, as the risks of ReDOS attacks could otherwise neutralise the benefits.
